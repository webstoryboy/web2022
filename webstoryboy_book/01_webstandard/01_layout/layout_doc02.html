소개

안녕하세요!. 웹스토리보이입니다. 사이트 만들기 위한 가장 기본적인 레이아웃 유형을 살펴보겠습니다.

이론

다음의 레이아웃을 작업할 것입니다. 레이아웃 방식은 크게 세가지가 있다고 했습니다.(float 방식, flex 방식, grid 방식) 이 방식을 하나씩 작업해보겠습니다. 즉 똑같은 레이아웃을 3가지 방법으로 작업해 볼 것입니다. 그럼 float 방식을 먼저 해보겠습니다. 처음 오시는 분들은 시작하기1를 참고하여, 에디터를 준비하시면 됩니다. 모든 분들이 처음 듣는다고 가정하게 기초적인 부분부터 차근 차근 설명하게 될 것이며, 보충이 필요한 부분은 참고 부분을 참고하시면 됩니다. 참고 사항이 아는 내용이라면 패스하시고 하나씩 하나씩 따라하시면 됩니다.

이 레이아웃은 사이트의 가장 기본적인 레이아웃으로 예전에 많이 사용하고, 지금도 여전히 사용하고 있는 float 방식으로 먼저 작업할 것입니다. 현재는 float보다는 flex를 더 많이 사용하는 추세지만, 예전에 만들어져 있던 사이트들은 float을 더 많이 사용했기 때문에 여러분들은 float 사용방법도 알고 있어야 합니다. 시간이 지나면 float을 사용 안 할 수도 있겠지만, 현재는 알아야 하기 때문에 공부해보도록 하겠습니다.

webstandard 폴더를 만들고 layout01.html 페이지를 만듭니다. 새로운 페이지를 열고 !를 치고 tab 버튼을 누릅니다. 그럼 기본 문서 소스가 나오고 여기에서 작업을 시작하겠습니다. 기본적으로 언어설정을 ko로 변경해주시고, 타이틀은 레이아웃01로 변경하겠습니다. 

<!DOCTYPE html>
<html lang="ko">
<head>
    <meta charset="UTF-8">
    <meta http-equiv="X-UA-Compatible" content="IE=edge">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>레이아웃01</title>
</head>
<body>
    
</body>
</html>

어렵지 않죠? 처음이니까 쉽게 쉽게 가겠습니다. 하나씩 조금씩 따라해주시면 됩니다. 하다가 안되는 부분이나 이상한 점이 있으면 댓글 달아주세요! 기본 셋팅이 끝나면 이제 레이아웃을 만들어 보도록 하겠습니다. 레이아웃 그림과 같이 사이트에서 레이아웃을 만들려면 우선 기본적으로 블록구조/인라인구조2의 개념을 알고 있어야 합니다. 이 개념을 잘 모른다면 아래 참고 부분을 보시고 오시면 되고요. 아는 분들은 지나가면 됩니다.

HTML 코드는 태그로 이루어져 있으며, 태그는 크게 블록 구조와 인라인 구조로 분류되며, 블록 구조는 박스, 인라인 구조는 텍스트라고 생각하시면 됩니다. 레이아웃은 박스를 쌓는 개념으로 이해하고, 그 안에 데이터들은 텍스트라고 생각하시면 됩니다. 그래서 블록구조로 레이아웃을 잡을 것이며, html 태그 중에 div 태그를 통해 레이아웃을 완성할 것입니다. 위에 완성된 디자인을 참고하면 5개의 박스로 구조가 이루어져 있으며, 제일 큰 박스 안에 5개의 박스가 들어가 있다고 가정할 것입니다. 그것을 코드로 표현해보겠습니다. 

<body>
    <div>
        <div></div>
        <div></div>
        <div></div>
        <div></div>
        <div></div>
    </div>
</body>

body 태그 안에 div 태그를 만들고 그 자식으로 5개의 div태그를 만들었습니다.

문서의 기본 뼈대는 완성했습니다. 간단하죠! 이제 살을 붙이겠습니다. 즉 디자인적 요소를 넣어주겠습니다. 그럴려면 CSS를 이용해야 합니다. style을 선언하고, 속성 값을 넣어주어야 합니다. 하지만 태그 이름이 다 똑같기 때문에 구별해주기 위해서는 id 값을 설정할 것입니다. 이것을 CSS 선택자라고 얘기하는데, 처음부터 자세히 들어가면 재미 없으니, 이것은 나중에 더 설명드리겠습니다. 지금은 하나씩 천천히 따라하는게 중요하고, 이해가 안가도 반복적으로 따라하다보면, 이해가 될 것입니다. 저를 믿고 한번 따라해 보세요! 😄 

<div id="wrap">
    <div id="header"></div>
    <div id="nav"></div>
    <div id="aside"></div>
    <div id="section"></div>
    <div id="footer"></div>
</div>

이렇게 태그마다 이름이 지워주면 됩니다. 즉 div는 본명, id는 가명이라고 생각하면 편할 것 같네요! 이렇게 이름이 다르면 각각 디자인 속성을 줄 수 있습니다.

<head>
    <meta charset="UTF-8">
    <meta http-equiv="X-UA-Compatible" content="IE=edge">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>레이아웃01</title>
    <style>
        #wrap {}
        #header {}
        #nav {}
        #aside {}
        #section {}
        #footer {}
    </style>
</head>

이렇게 head 태그 안에 설정할 수 있으며, 여기에 속성 값, 즉 디자인적 요소를 넣어주면 됩니다. 그래서 영역을 잡고 배경색을 넣어주면 레이아웃 구성을 확인 할 수 있습니다. 

<style>
    #wrap {
        width: 1200px
    }
    #header {
        width: 1200px;
        height: 100px;
        background-color: #FFE0B2;
    }
    #nav {
        width: 1200px;
        height: 100px;
        background-color: #FFCC80;
    }
    #aside {
        width: 400px;
        height: 780px;
        background-color: #FFB74D;
    }
    #section {
        width: 800px;
        height: 780px;
        background-color: #FFA726;
    }
    #footer {
        width: 1200px;
        height: 100px;
        background-color: #FF9800;
    }
</style>

이렇게 width값, height값, background-color을 넣어주면 영역이 잡힙니다. 이 부분을 확인하려면 Live Server 확장기능(extensions)3을 이용하시면 됩니다. 확인하는 방법을 잘 모르면 해당 번호 참고를 확인하시면 됩니다.

여기까지 결과화면 확인하기 (layout_result01_05.html)

결과 화면을 확인해 보면 박스들이 세로로 정렬되어 있는 것을 확인 할 수 있습니다. 블록구조는 한줄에 하나씩 나오는 성질 때문에 가로로 정렬되지 않고, 세로로 정렬되어 있습니다. 이 부분을 정렬시켜 주기 위해서 #aside와 #section 요소에게 float:left;를 주어 가로로 정렬시키겠습니다. 

<style>
    #aside {
        width: 400px;
        height: 780px;
        background-color: #FFB74D;
        float: left;
    }
    #section {
        width: 800px;
        height: 780px;
        background-color: #FFA726;
        float: left;
    }
    #footer {
        width: 1200px;
        height: 100px;
        background-color: #FF9800;
        clear: both;
    }
</style>

이렇게 하면 가로로 정렬되지만, float으로 인하여 버그4가 발생합니다. 이런 버그가 발생하는 이유는 원래 float은 레이아웃을 위한 속성이 아니기 때문에 그렇습니다. 즉 float 속성을 사용하게 되면 영역의 height 값이 0으로 되기 때문에 footer 박스는 윗쪽으로 올라가서 보이지 않게 됩니다. 그래서 footer는 사라지는 것 처럼 보이고, 이것을 방지하기 위해 푸터 영역에 clear:both를 주면 이런 문제점을 해결 할 수 있습니다.

여기까지 결과화면 확인하기 (layout_result01_06.html)

그리고 마지막으로 전체 영역 #wrap 요소에게 margin: 0 auto를 사용하여 가운데 정렬을 하면 됩니다. 가운데 정렬 하는 방법은 블록구조 가운데 정렬5를 참고하면 됩니다. 그리고 화면을 보면 윗쪽에 여백이 생기는 것을 확인 할 수 있습니다. 이건 모든 요소들이 기본적으로 패딩이나, 마진값을 가지고 있어서 그렇습니다. 그래서 따로 설정하지 않아도 저렇게 간격이 생깁니다. 이 간격들이 브라우저마다 똑같으면 상관없지만, 다르기 때문에 초기화가 필요합니다. 그래서 전체 선택자(*)를 이용해서 여백을 없애것입니다. 

* {
    margin: 0;
    padding: 0;
}

여기서 모든 블록구조6의 박스는 안쪽 여백(padding), 보더값(border), 바같쪽 여백(margin)을 가지고 있습니다. 이것에 대한 설명은 참고를 확인하시면 됩니다.

전체 선택자(*) 역시 id 값을 정해주는 선택자의 일종입니다. 선택자는 앞으로도 다양하게 나오기 때문에 지금은 "이런거다" 하고 넘어가겠습니다. 처음부터 너무 진지하게 알려주면, 재미없으니, 차후에 반복적으로 더 알려드리겠습니다. 😙
