소개

안녕하세요!. 웹스토리보이입니다. 이번 주제는 이미지 유형을 알아보겠습니다. 이미지 유형은 사이트 제작에 있어 많이 사용되는 유형입니다. 마우스 오버시 정보가 제공되는 형태입니다. 그럼 바로 시작하겠습니다.

이론

이번에 만들어 볼 유형은 이미지 유형입니다. 2단 구조로 되어 있는 간단한 구조입니다. 평소에는 제목만 있고, 마우스 오버시 설명과 버튼이 나오는 구조입니다. 그럼 바로 시작하겠습니다. 우선 image_code01.html 페이지를 만들고, !를 치고 tab을 누르겠습니다. 언어를 ko로 변경하고 title은 카드 유형01로 변경하겠습니다. 

<!DOCTYPE html>
<html lang="ko">
<head>
    <meta charset="UTF-8">
    <meta http-equiv="X-UA-Compatible" content="IE=edge">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>이미지 유형01</title>

    <link href="https://webfontworld.github.io/gmarket/GmarketSans.css" rel="stylesheet">

여기까지 결과 확인하기

section을 만들고, 가운데 구조 container를 만듭니다. section 속성에는 padding: 120px 0을 설정하고 전체 선택자(*)를 이용하여, 모든 여백을 초기화합니다. 가운데 영역을 설정하기 위해 클래스 .container를 설정합니다.

제목과 설명을 표현하기 위하여 h2 태그와 p 태그를 사용하여, 내용을 설정합니다. .h2, .desc 클래스를 넣어주고, 폰트 사이즈와 간격 등을 설정합니다. 지마켓 웹폰트1를 설정합니다.

이번에는 이미지를 추가하고 레이아웃을 잡겠습니다.

<div class="image-inner">
    <div class="image">
        <img src="../../assets/img/imagetype01.jpg" alt="애교만점 웰시코기">
    </div>
    <div class="image">
        <img src="../../assets/img/imagetype02.jpg" alt="코카스 파니엘">
    </div>
</div>

div 박스를 하나 만들고 자식으로 2개의 div 박스를 넣고, 클래스 image-inner와 image를 추가하였습니다. img 태그를 사용하여 이미지를 출력하고 alt 속성을 추가하였습니다. 

.image-inner {
    display: flex;
    justify-content: space-between;
}
.image-inner .image {
    width: 49%;
}
.image-inner .image img {
    width: 100%;
}

.image-inner에 flex를 선언하고, .image 클래스에 width 값을 넣어줍니다. 이미지는 width에 100%를 선언하여, 박스 크기에 맞추어 줍니다.

여기까지 결과 확인하기

image-text 박스를 만들고 h3, p, a 태그를 이용하여, 내용물을 채워줍니다. 두 개의 이미지 박스에 동일하게 작업합니다. 

.image-inner .image {
    width: 49%;
    position: relative;
    text-align: left;
}
.image-inner .image img {
    width: 100%;
}
.image-inner .image .image-text {
    position: absolute;
    left: 30px;
    bottom: 30px;
    color: #fff;
}
.image-inner .image .image-text h3 {
    font-size: 32px;
    font-family: 'NexonFootballGothic';
    font-weight: 300;
    margin-bottom: 10px;
}
.image-inner .image .image-text p {
    font-size: 16px;
    font-family: 'NexonLv1Gothic';
    font-weight: 400;
    line-height: 140%;
    margin-bottom: 16px;
}
.image-inner .image .image-text a {
    text-decoration: none;
    color: #fff;
    font-family: 'NexonLv1Gothic';
    background: #BE4E15;
    padding: 10px 20px;
    display: inline-block;
}
.image-inner .image:last-child .image-text a {
    background: #C9931D;
}

.image-text 클래스에 position:asbolute를 설정합니다. asbolute를 설정 했기 때문에 기준점 역할을 하는 relative를 상위 박스 .image 클래스에 설정합니다. 제목, 설명, 버튼에도 해당 속성을 설정합니다.

버튼은 inline 구조이기 때문에 패딩이나 마진이 안 먹힐 수 있어서 inline-block을 넣어줍니다. 이렇게 하면 영역도 먹히면서 가로 정렬이 됩니다.

여기까지 결과 확인하기

이번에는 웹 표준에 맞게 시멘틱 태그를 사용하고 네이밍도 다시 설정하겠습니다. <article> 태그는 사이트의 독립적인 컨텐츠 섹션을 설정할 때 사용하므로 이미지 섹션은 <article> 태그로 설정하겠습니다. <figure> 태그는 이미지를 표현 할 때 사용하므로 <img> 태그와 같이 사용하겠습니다. 전체적인 구조를 image__header와 image__body로 나누어서 좀 더 체계적으로 설정하였고, 네이밍도 알기 쉽게 체계화하였습니다.

여러분들도 이렇게 코딩을 다시 한번 해보시고 저랑 비교해 보면 좋을 것 같습니다. 클래스 이름도 부모-자식 관계로 써도 좋지만, 하이픈(__)을 사용하여 쓰는 방법도 있습니다. 클래스 이름 표현하는 방법도 여러가지가 있지만, 여러분 나름대로 효과적이고, 효율적인 방법으로 사용하면 됩니다. 또는 회사에서 쓰는 규칙이 있다면 그 방법을 사용해도 됩니다.

웹 표준 코딩에 있어 의미에 맞는 태그 사용도 중요하지만, 누구나 알아보기 쉽게 적절한 네이밍을 사용하는 것도 매우 중요합니다. 별거 아닌 것 같지만 이런 사소한 부분부터 신경을 써주는게 웹 표준, 웹 접근성의 기본이라고 할 수 있습니다.
