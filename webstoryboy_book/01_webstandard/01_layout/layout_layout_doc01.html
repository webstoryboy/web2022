레이아웃 유형 : 레이아웃01

난이도 쉬움

소스 다운로드

완성 소스 완성 디자인 피그마

소개

안녕하세요! 웹스토리보이입니다. 웹표준 사이트를 만들기 위한 가장 기본적인 단계로 레이아웃을 만들어보겠습니다. 레이아웃은 사이트를 만들기 위한 아주 기본적인 단계입니다. 모든 사이트는 레이아웃 속에 레이아웃으로 구성되어 있습니다. 레이아웃을 완벽하게 이해하면 사이트의 반은 끝났다고 봐도 됩니다.

레이아웃을 제작하는 방법은 여러가지가 존재합니다. 90년대만 하더라도 레이아웃은 테이블로 구조를 잡고 코딩을 하였습니다. 하지만 이 방법은 웹 표준을 준수하지 못한 방법으로 현재는 사용하고 있지 않습니다. 그 다음으로 사용하는 방법은 float을 이용한 방법입니다. 이 방법도 레이아웃을 잡기 위한 속성은 정확히 아니기 때문에 이 때문에 어쩔 수 없이 사용하였습니다. 레이아웃을 표현하는 속성이 없었기에 버그가 있지만 이 방법으로 대부분의 사이트가 사용하게 되었습니다.

웹 표준과 웹 접근성이 강조가 되고 레이아웃의 중요성이 높아지면서 flex와 grid 속성을 사용하게 되었습니다. 현재 22년을 기준으로 float 보다는 flex의 사용성이 많은 것 같습니다. 시간이 흐르면 grid의 사용성이 높아 질 수도 있지만, 저희는 이 세가지 방법을 사용하여 레이아웃을 작업 할 것입니다.

처음하시는 분들은 모든 것이 어색하고 낯설을 수 있습니다. 하지만 그냥 아무생각 없이 따라하다 보면, 반복적으로 연습하다 보면, 어느 순간 사이트를 만드는 자신을 볼 수 있을 겁니다. 중요한 건 수도 없이 반복되고, 저절로 외워지고, 머릿 속에 개념이 잡힐 것입니다. 모른다고, 이해 안된다고, 멈출 필요 없습니다. 그냥 따라하시면 됩니다. 믿고 따라와보세요! 그럼 끝까지 잘 따라해 봅시다. 그럼 시작해볼까요?

이론

그럼 vscode를 키고 layout01.html 페이지를 만들겠습니다. 왼쪽에 살며시 보이는 숫자는 완성 코드의 넘버니까 작업할 때 참고하세요! 새로운 페이지에 !를 누르고 Tab을 누르면 기본 웹 문서의 기본 양식이 나옵니다. 이 기능은 emmet 기능으로 vscode에 기본으로 내장되어 있어, 자동으로 됩니다. 만약 이 부분이 안된다면 본인이 만든 파일이 .html로 끝나는지 확인해보세요! html 파일에서만 작동되는 기능입니다.

<body>

다음과 같이 코드가 나오면 성공한 겁니다.

사이트를 처음 만들어 본다고 가정하고, 처음에는 하나씩 설명을 드리겠습니다. 물론 아는 부분은 스킵하셔도 상관없습니다. 처음부터 너무 자세하게 설명하면, 설명이 무거워지고 재미가 없으니, 가볍게 가볍게 설명하다가 점점 깊게 공부하는 방향으로 진행이 될 것입니다.

브라우저가 인식 할 수 있는 언어를 설정하는 부분입니다. 우리는 ko로 변경할 겁니다. 1

웹 문서는 head와 body로 나누어지며, head 안에는 사이트에 정보 등을 표현합니다.

이 부분은 컨텐츠의 언어를 설정하는 부분입니다. 2

웹 문서를 만들기 위한 기본적인 규칙을 설정하였습니다. 처음엔 가볍게 설명하고, 중요한 건 반복적으로 설명하여, 저절로 알 수 있게 공부할 것입니다. 자세히 알고 싶으면 자세히 알아보기를 눌러서 확인하셔도 됩니다. 그럼 본격적으로 시작해 보겠습니다.

레이아웃을 만들기 위해서는 우선 Block 구조와 Inline 구조를 6 이해해야 합니다. 태그는 150가지 정도가 있는데, 크게 블록구조와 인라인구조로 분류합니다. 블록은 박스개념, 인라인은 텍스트 개념으로 이해하시면 됩니다. 즉 레이아웃을 만들려면 블록구조의 대표주자 div 태그를 사용합니다. 물론 section 태그를 사용할 수 있지만, 이건 나중에 배울겠습니다. 

<body>
    <div id="container">
        <div id="header"></div>
        <div id="nav"></div>
        <div id="aside"></div>
        <div id="contents"></div>
        <div id="footer"></div>
    </div>
</body>

body태그 안에 div 태그를 만들고 그 안에 div 태그를 5개 만들 것입니다. 태그 이름이 같기 때문에 구별해주기 위해서 각각에 이름을 설정해 줄 것입니다. id를 설정하여, 구분하겠습니다. 이름을 설정할 때는 id와 class를 사용할 수 있으며, 우리는 id를 사용하여, 작업해 보겠습니다. 이렇게 작성하여도 브라우저에는 아무것도 나타나지 않습니다. 디자인 작업이 되어 있지 않기 때문에, 디자인 역할을 하는 css를 설정하겠습니다. 

<style>
    body {
        background-color: #FFF3E0;
    }
    #container {
        width: 1200px;
        height: 1080px;
    }
    #header {
        width: 1200px;
        height: 100px;
        background-color: #FAE1B7;
    }
    #nav {
        width: 1200px;
        height: 100px;
        background-color: #F7CE8B;
    }
    #aside {
        width: 400px;
        height: 780px;
        background-color: #F4BA61;
    }
    #contents {
        width: 800px;
        height: 780px;
        background-color: #F2AB47;
    }
    #footer {
        width: 1200px;
        height: 100px;
        background-color: #F19D38;
    }
</style>

이렇게 width, height을 넣어주면, 영역이 잡히고, 구별을 해주기 위해서 background를 넣어주면 영역을 확인할 수 있습니다.
